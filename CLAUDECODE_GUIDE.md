# Claude Code への指示ガイド（非エンジニア向け）

このプロジェクトを GitHub に Push（アップロード）するための、Claude Code への指示文です。
コピペして使ってください。

---

## 事前準備（あなたがやること・1回だけ）

### 1. リポジトリを展開する
ダウンロードした `football-chess-repo.tar.gz` を、デスクトップなどわかりやすい場所に展開します。
- Mac: ダブルクリックで展開できます
- Windows: 右クリック →「すべて展開」、または 7-Zip 等で展開

展開すると `football-chess-prototype.html` や `HANDOFF.md` が入ったフォルダができます。

### 2. Claude Code をそのフォルダで起動する
ターミナル（Macは「ターミナル」、Windowsは「PowerShell」）を開き、展開したフォルダに移動してから `claude` と打って起動します。

```bash
# 例（フォルダ名は実際のものに置き換え）
cd ~/Desktop/football-chess-repo
claude
```

> フォルダへの移動が難しい場合は、Claude Code 起動後に「いまどのフォルダにいる？」と聞けば教えてくれます。

## Claude Code に貼る指示文（メイン）

下の枠の中をそのままコピーして、Claude Code に貼り付けてください。
GitHub のユーザー名とリポジトリ名だけ、自分のものに書き換えてください。

```
あなたは私（非エンジニア）の代わりに、このフォルダの Git リポジトリを GitHub に push する作業を手伝ってください。一歩ずつ、私にわかる言葉で説明しながら進めてください。

前提:
- このフォルダはすでに git でコミット済みです（コミット履歴あり）。
- これから GitHub のリモートリポジトリに push したいです。

やってほしいこと:
1. まず `git status` と `git log --oneline` を実行して、現在の状態（コミット済みか、何件コミットがあるか）を私に説明してください。
2. GitHub に push するために必要な手順を、私が GitHub 側で何をすべきか（リポジトリ作成、トークン発行など）も含めて、順番に教えてください。
3. リモート URL を登録する `git remote add` のコマンドを、私が入力する GitHub ユーザー名・リポジトリ名を聞いたうえで作ってください。
4. push まで実行してください。途中で認証（ユーザー名・トークン）が必要になったら、どこで何を入力すればいいか教えてください。
5. push が成功したら、GitHub のどの URL で確認できるか教えてください。

注意:
- 私はコマンドの意味がわからないので、実行する前に「これは何をするコマンドか」を一言添えてください。
- エラーが出たら、原因と次にすべきことをやさしく説明してください。
```

## もし「既存のリポジトリに入れたい」場合

すでに `yanagiho/football-chess-next` という GitHub リポジトリがあり、そこに成果物を入れたい場合は、
上の指示の代わりに下を貼ってください。

```
私（非エンジニア）の代わりに作業してください。一歩ずつ、わかる言葉で説明しながら進めてください。

状況:
- このフォルダに football-chess-prototype.html などの成果物があります。
- 既存の GitHub リポジトリ「yanagiho/football-chess-next」に、この成果物を追加したいです。
- 履歴の衝突を避けたいので、既存リポジトリを別の場所にクローンして、そこに成果物をコピーして push する方法でお願いします。

やってほしいこと:
1. 既存リポジトリをクローンするコマンドを教えて、実行してください。
2. この成果物ファイル（football-chess-prototype.html, HANDOFF.md など）を、クローンしたリポジトリ内の適切なフォルダにコピーしてください。フォルダ構成に迷ったら私に確認してください。
3. コミットして push してください。認証が必要なら、どこで何を入力するか教えてください。
4. 成功したら確認用の URL を教えてください。

私はコマンドの意味がわからないので、実行前に一言で説明を添えてください。エラーが出たらやさしく説明してください。
```

## GitHub のトークン（パスワード代わり）について

push のときに「Username」と「Password」を聞かれます。Password には、ふだんのログインパスワードではなく
**Personal Access Token（PAT）** を入れます。発行場所:

GitHub にログイン → 右上のアイコン → Settings → 一番下の Developer settings
→ Personal access tokens → Tokens (classic) → Generate new token

- 用途（repo）にチェックを入れて発行
- 発行されたトークン文字列をコピーして、Password 欄に貼り付け

> トークンは発行時の1回しか表示されないので、メモしておくと安心です。
> Claude Code にトークン文字列そのものを貼る必要はありません（ターミナルの認証画面に入れます）。

## うまくいかないとき

- Claude Code が出したエラーメッセージをそのまま Claude Code に「これは何？どうすればいい？」と聞けば対応してくれます。
- それでも詰まったら、エラー全文をコピーして、また私（このチャット）に貼ってください。一緒に解決します。
