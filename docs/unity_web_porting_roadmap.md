# Football Chess ManiacS — Unity版踏襲Web移植ロードマップ

作成日: 2026-06-28

## 1. 目的

Football Chess ManiacS は、Unity版 Football Chess を基本方針として踏襲しつつ、Web/PWA と Cloudflare サーバー基盤で再構成する。

本ロードマップでは、以下を開発方針として固定する。

- 新しいGitHubリポジトリを作成し、既存FCMSとは別プロジェクトとして運営する
- 基本的なゲーム体験・ルール・世界観は Unity版 Football Chess を正典として扱う
- 実装基盤は Web フロントエンド + Cloudflare Workers / Durable Objects / D1 を中心にする
- 広告モデルは採用しない
- 収益化はサブスク課金を基本にする
- COM AI、マッチング、ランキングを現代的なWeb版の強化ポイントにする
- 軽い継続アップデートを前提に、運用しやすいデータ構造にする

## 2. プロジェクト運営方針

このWeb移植版は、既存の `football-chess-maniacs` を直接改造し続けるのではなく、新規GitHubリポジトリで別プロジェクトとして進める。

既存FCMSは以下の用途で参照する。

- 既に実装済みのEngine / AI / UI / Cloudflare構成の部品取り
- Unity版ルール調査結果の参照
- NPCチーム、世界観、アセット候補の参照
- 既存の課題、操作性改善、テスト資産の移植元

新規プロジェクト側では、初期段階で以下を明確に分ける。

- Unity版から移植する正典仕様
- 既存FCMSから再利用する実装
- Web版として新規設計する部分
- いったん捨てる既存FCMS仕様

推奨する新規リポジトリ名候補:

- `football-chess-web`
- `football-chess-online`
- `football-chess-next`
- `football-chess-maniacs-next`

以後、この文書では新規リポジトリ側を「Web移植版」と呼ぶ。

## 3. 最重要判断: Unity版をどこまで踏襲するか

「Unity版を踏襲」には2段階ある。

### A. 体験・ルールを踏襲する

現在の Football Chess ManiacS のHEX盤面・React UIを維持しつつ、以下をUnity版に寄せる。

- オフサイド、ファウル、遅延行為、セットプレーなどのサッカールール
- 試合テンポ、演出、UI上の気持ちよさ
- NPCチーム、世界観、チーム性格
- COMの戦術傾向

短期で現実的。既存実装を活かせる。

### B. 盤面・操作・ルール構造までUnity版に戻す

Unity版のセル構造、操作感、判定順、ゲームテンポをWeb上で再現する。

真の移植に近いが、現在のHEXエンジン・UIとの設計差が大きいため、かなり大きな作り直しになる。

### 推奨

まずは A を採用する。Unity版の優れたルール・試合感・世界観を現在のFCMSに取り込み、操作性とWeb運用を磨く。

ただし、今後「やはりUnity版そのものの盤面が良い」と判断した場合は、B を別プロジェクト級の再設計として扱う。

## 4. プロダクト方針

### ゲーム本体

- Unity版のサッカーチェスらしさを正典にする
- Web版ではスマホ操作を最初から重視する
- 操作説明を読ませるのではなく、コマ周辺のコマンド、選択演出、ハイライトで理解できるUIにする
- ルールは複雑でも、画面上の判断は軽くする

### サーバー

- Cloudflare Workers: REST API / 認証後API / 軽量処理
- Durable Objects: ゲームセッション、リアルタイム対戦、マッチング状態
- D1: ユーザー、チーム、試合結果、ランキング、サブスク権利
- R2: リプレイ、画像、将来的な大きめアセット
- KV: JWKSキャッシュ、短期キャッシュ、レート制限補助
- Queues / Cron: ランキング集計、シーズン更新、非同期ログ処理
- Workers AI: COM AIの推論候補

Cloudflareの具体的な制限・料金・最新APIは実装前に公式ドキュメントで確認する。

### 収益化

広告は入れない。サブスク課金を基本にする。

推奨するサブスク特典は、対戦公平性を壊さないものに寄せる。

- 編成セーブスロット追加
- リプレイ保存数の増加
- 高度なCOM対戦、特殊NPCチャレンジ
- シーズン称号、プロフィール装飾、コスメティック
- プライベートルームや大会補助機能
- 詳細戦績、棋譜分析、AIアドバイス

避けたいもの。

- ランクマッチで強いコマを課金限定にする
- 課金者だけ判定有利にする
- 無課金では対戦が成立しにくい導線にする

既存仕様にはコマ購入・ショップ要素が残っているため、サブスク中心へ寄せるなら課金仕様を再整理する必要がある。

## 5. 主要機能ロードマップ

### 5-1. Unity版踏襲

対象:

- オフサイド判定
- ファウル判定
- 遅延行為
- セットプレー
- ボール保持、ルーズボール、スペースへのパス
- NPCチームの世界観
- 試合開始、ゴール、ハーフタイム、フルタイム演出

進め方:

1. Unity版ソースのルールを一覧化する
2. 現FCMS実装との差分を分類する
3. 「そのまま移植」「Web向けに調整」「採用しない」を決める
4. Engine側にテストを追加してから実装する

### 5-2. COM AI強化

方針:

- ルールベースAIを安全層にする
- 合法手生成と違法手排除は必ずコード側で行う
- Workers AI / Gemma系の推論は判断層として使う
- 推論失敗時は必ずルールベースにフォールバックする

強化項目:

- ボール保持者の選択精度
- スペースへのパス
- オフサイドラインの理解
- 守備ブロックとZOC活用
- NPCチームごとの戦術性格
- 難易度 beginner / regular / maniac の差別化
- 自動対戦による品質検証

### 5-3. マッチング強化

モード:

- Casual: 早く遊べる
- Ranked: レーティング近接を優先
- Private: 友達対戦、URL共有
- COM: 人がいない時でも遊べる

強化項目:

- レート帯による候補探索
- 待ち時間に応じた探索範囲拡大
- 地域/レイテンシを考慮したシャード
- 切断時の復帰
- 放置・遅延・不正対策
- 人が見つからない場合のCOM代替

### 5-4. ランキング追加

ランキング種別:

- シーズンランキング
- 通算ランキング
- 直近ランキング
- 自分周辺ランキング
- NPCチャレンジランキング

必要データ:

- 試合結果
- レーティング変動
- 勝敗、得点、失点
- 切断、無効試合
- シーズンID

注意:

- ランキングは試合結果APIだけで即時更新せず、D1更新と非同期集計を分ける
- 不正対策のため、サーバー側で確定した試合結果のみ反映する

### 5-5. 軽いアップデート

継続更新で扱いやすいもの:

- NPCチーム追加
- 期間限定チャレンジ
- コスメティック
- バランス調整
- UI改善
- 演出追加
- 翻訳追加
- チュートリアル改善

重くなりやすいもの:

- 盤面ルール変更
- コスト体系変更
- 課金アイテムの性能変更
- ランクマッチ仕様変更

軽いアップデートを続けるため、ルール・課金・ランキングの根幹は早めに固定する。

- オフサイド、ファウル、遅延行為、セットプレーをEngineに統合
- ユニットテストを先に増やす

### Phase 5: ランキング
